1) В форме регистрации отсутствует согласие  обработку персональных данных.
Шаги воспроизведения:
1. Переходим на страницу http://localhost:5000/signup
2. Проверям форму регистрации на наличие чекбокса с согласием на обработку персональных данных.
Ожидаемый результат: чекбокса с согласием на обработку персональных данных должен быть на странице регистрации.

2) Существует возможность создать пустой аккаунт без заполненных полей Email, password, name.
Шаги воспроизведения:
1. Переходим на страницу http://localhost:5000/signup
2. Нажимаем на кнопку signup(поля оставляем пустыми)
Ожидаемый результат: при нажатии на кнопку signup должно появлятся уведомление о необходимости заполнения обязательных полей.

3) Существует возможность зайти в созданный пустой аккаунт.
Шаги воспроизведения:
1. Переходим на страницу http://localhost:5000/login
2. Нажимаем на кнопку Login(поля и чекбокс оставляем пустыми)
Ожидаемый результат: при нажатии на кнопку логина должно появлятся уведомление о необходимости заполнения обязательных полей.

4) Существует возможность создать аккаунт с одним заполненным полем Email.
Шаги воспроизведения:
1. Переходим на страницу http://localhost:5000/signup
2. Корректно заполняем поле Email.
3. Нажимаем на кнопку signup.
Ожидаемый результат: при нажатии на кнопку signup должно появлятся уведомление о необходимости заполнения обязательных полей(password).

5) Отсутствуют ограничения на количество букв в полях Email, password, name при регистрации аккаунта.
Шаги воспроизведения:
1. Переходим на страницу http://localhost:5000/signup
2. Вводим в поле Email рандомный почтовый ящик с количеством букв больше 20(например).
3. Вводим в поле password рандомный пароль с 1 буквой или символом.
4. Нажимаем на кнопку signup.
Ожидаемый результат: поля Email, password, name должны содержать ограничения на количество вводимых букв(например от 5 до 20).

6) Существует возможность заполнить поля password, name исключительно с помощью символов.
Шаги воспроизведения:
1. Переходим на страницу http://localhost:5000/signup
2. Вводим в поле password рандомный пароль использую исключительно символы (например !@#).
3. Нажимаем на кнопку signup.
Ожидаемый результат: Поле name не должно содержать символов !@#. Поле password должно содержать заглавные и строчные буквы.

7) Существует возможность заполнить поля password, name исключительно с помощью символов.
Шаги воспроизведения:
1. Переходим на страницу http://localhost:5000/signup
2. Вводим в поле password рандомный пароль использую исключительно символы (например !@#).
3. Нажимаем на кнопку signup.
Ожидаемый результат: Поле name не должно содержать символов !@#. Поле password должно содержать заглавные и строчные буквы.

8) На странице логина чекбокс Remember me ни на что не влияет.
Шаги воспроизведения:
1. Переходим на страницу http://localhost:5000/login
2. Активируем чекбокс Remember me.
3. Заходим в аккаунт.
4. Выходим из аккаунта.
5. Анализируем результат.
Ожидаемый результат: В случае активации чекбокса "Remember me" поля email, password должы автоматически заполнятся в случае выхода из аккаунты и при поторном заходе в него.
